================================================================================
START IMPORT EXPORT — PROJECT + UI CONTEXT
================================================================================

A Flutter (Dart) mobile app for SIIEA (Startup India Import Export Academy) — an
ed-tech + business-tools platform that teaches Indian entrepreneurs how to start
and run an import-export (EXIM) business.

  Package / applicationId : com.startimportexport.app
  Launcher activity       : com.startimportexport.app.MainActivity
  Platforms               : Android + iOS only (windows/linux/macos removed)
  Backend API             : http://52.35.230.229:3006
  Auth                    : phone + OTP -> JWT Bearer (attached automatically)

A5. NEWS + NOTIFICATION DEEP-LINK FLOW
--------------------------------------------------------------------------------
News details are ALWAYS opened by id — the screen fetches fresh data itself, so
the list can never be stale.

      Push notification tap ──┐
      In-app notification  ───┼──►  newsId  ──►  NewsDetailScreen(newsId:)
      News list tap        ───┘                        │
                                                       ▼
                                        GET /api/news/:id
                                        -> render  |  404 -> error + Retry

  BACKEND PAYLOAD CONTRACT
    FCM push:
        notification: { title, body }          <- what the user sees in the tray
        data:         { type: "news",
                        newsId: "<mongo _id>" } <- used for routing (REQUIRED —
                                                   tray text alone is not enough
                                                   in background/killed state)
    In-app notification item (GET /api/notifications):
        data:    { type: "news", newsId: "<id>" }   (preferred)
        linkUrl: "news://<id>"                      (fallback)
        source:  "NEWS_AUTO"

  WHERE IT'S WIRED
    core/services/notification_router.dart
        Global navigatorKey, routeFromData(), news:// parser,
        AwesomeNotifications tap listener
    core/services/firebase_messaging_service.dart
        onMessage        -> foreground: build local notification (carries data)
        onMessageOpenedApp -> background tap
        getInitialMessage  -> killed-state tap (routed ~4s after splash)
    main.dart
        MaterialApp(navigatorKey: NotificationRouter.navigatorKey) + setListeners

  News list uses SERVER-SIDE PAGINATION — next page loads at scroll-end
  (NewsProvider.loadMore(), guarded against duplicate calls, bottom spinner).


--------------------------------------------------------------------------------
A6. AI ASSISTANT (lib/features/chatai) — base path /api/assistant
--------------------------------------------------------------------------------
  GET  /starters              Starter chips + hasOldChats flag
  POST /chat                  Send message. OMIT conversationId on the FIRST send;
                              include it on every message after that.
  GET  /conversations         Old-chats list
  GET  /conversations/:id     Full thread

  RULES
    - Always opens on a fresh "New chat" (never the last conversation)
    - Starter chips show only while the thread is empty
    - "Old chats" button appears only when hasOldChats == true
    - Optimistic user bubble + animated typing dots
    - Send disabled while sending (blocks duplicate API calls)
    - Max message length 4000
    - Errors: 503 = AI not configured, 502 = OpenAI failed -> snackbar + Retry
    - Long-press any bubble to copy

--------------------------------------------------------------------------------
A9. CONVENTIONS (follow these)
--------------------------------------------------------------------------------
  - New endpoint?  Add it to ApiConstants, call it through callApi.
  - New colour?    Add it to AppColors — do not scatter hex codes.
  - Any API-triggering button MUST show a loader and be disabled while in flight
    (prevents duplicate calls). Use AsyncButton, or the provider's
    isLoading / sending / isSyncing flag.
  - Localisation: every user-facing string goes in lib/localization/l10n/en.json
    AND hi.json; read with AppLocalizations.of(context).translate('key').
  - Timers/streams MUST stop in deactivate() / dispose() and when the app goes to
    background. A running PageController.animateToPage on a deactivated widget
    crashes the app (this happened with the masterclass carousel).
  - Run `flutter analyze` before committing. The project is kept at ZERO issues.

      flutter analyze lib    (must be clean)

  enough when testing notifications. Do a full run.

--------------------------------------------------------------------------------
B1. DESIGN PRINCIPLES (came directly from the client — follow strictly)
--------------------------------------------------------------------------------
  1. LIGHT THEME BY DEFAULT. Screens are white. The app is not a dark app.
  2. CARDS MUST NOT BLEND INTO THE BACKGROUND. White cards on a white page are
     fine ONLY with a soft shadow to lift them. "Halki shadow kaafi hai."
  3. DARK ELEMENTS ON LIGHT BACKGROUND LOOK GOOD. Do not wash everything to white
     — a dark premium card floating on a white page is the intended look.
  4. NO FIXED HEIGHTS ON CARDS. A card takes only the space its content needs.
     Horizontal lists use SingleChildScrollView + Row (not a fixed-height
     ListView); media uses AspectRatio so height follows width.
  5. LIMITED COLOUR PALETTE (see B2). No rainbow gradients. Tool/feature icons use
     a tinted square + a single brand colour.
  6. MATERIAL ICONS ONLY. The old custom CustomPainter icon set was removed.
  7. MINIMAL ANIMATION. One subtle page fade per screen. No per-widget bounce
     chains (login/OTP originally had 9 staggered animations — all removed).
  8. TEXT MUST BE READABLE. Font sizes were raised twice on client feedback.
     Current ladder is in B3.

--------------------------------------------------------------------------------
B7. UI PITFALLS ALREADY HIT — DO NOT REPEAT
--------------------------------------------------------------------------------
  - White text left on a widget that later moved onto a white background became
    invisible (this happened with the Gallery marquee heading). When you change a
    surface from dark to light, flip EVERY text/icon colour inside it.
  - A fixed-height horizontal ListView clipped card content. Use
    SingleChildScrollView + Row so the height follows the tallest child.
  - A Timer kept calling PageController.animateToPage after the widget was
    deactivated -> "Looking up a deactivated widget's ancestor" crash. Stop timers
    in deactivate() and on app background; mounted alone is NOT sufficient.
  - Firebase shows nothing by itself in the FOREGROUND — you must build a local
    notification from onMessage, or the user sees no notification while the app is
    open.
